MFDC Project Methodology (13)

MFDC use a stage gate project management methodology and will not deviate from this approach unless agreed by the customer and Director MFDC. The MFDC Project Methodology is as follows:

Stage Description Detail
1 Quoted Estimate The estimated price and breakdown of service from MFDC and any relevant 3rd party contractors.
2 Scope and Technical Research and Analysis The initial outline of the project capturing functional and non-functional requirements.
3 Prototype (Technical Design and Wireframes) Non-functional review of the system to provide the client with an idea of the technical design and an opportunity to provide early feedback.
4a System Build (incl. Alpha Release) The build stage is where the core of the system and database are constructed. A functional working prototype of the system will provide the client with an opportunity to use the system in an 80% complete working state to allow for clear understanding of the development direction and how business requirements are being met.
4b System Build (incl. Beta Release) This is where all feedback is incorporated into the system and is declared ready for user acceptance testing. The system is 100% complete and will not entertain any more variation or changes.
4c System Integration (if required) This is not a standard stage and is used if required. This is where the system has a requirement to integrate with 3rd party products.
5 Client User Acceptance Testing This is where the client reviews and fully tests the system against the Acceptance Criteria as if it is ready to go into production. The client will provide feedback on any known performance or functional issues. This stage does not introduce any new functionality to the application unless agreed by a variation.
6 Application Packing and Handover MFDC will package the system ready for hosting or deployment and provide this to the customer.

Deliverables for each stage as outlined above must be completed before commencement onto subsequent stages. The deliverables of each stage is as follows:

Stage Description Deliverables
1 Quoted Estimate Quoted estimate must be signed by both parties for project commencement.
2 Scope and Technical Research and Analysis Scope document is to be completed using the MFDC template; this may include technical specifications if required. User Acceptance Testing criteria begins development once both parties have agreed on the scope.
3 Prototype (Technical Design and Wireframes) Wireframe or low level design. If required, wireframe or design documentation will be performed before prototype can begin. This stage also delivers a non-functional prototype.
4a System Build (incl. Alpha Release) 80% of all functionality has been developed for the system.
4b System Build (incl. Beta Release) 100% of all functionality has been completed.
5 Client User Acceptance Testing Client and MFDC perform tests against criteria. MFDC performs bug fixes.
6 Application Packing and Handover A handover document is delivered. Once signed, the application is packaged and provided to the client.
